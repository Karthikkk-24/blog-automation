================================================================
  KEY 1: GEMINI_API_KEY
  Service : Google Gemini (writes your entire blog)
  Cost    : FREE — 1,500 requests/day on free tier
  Time    : ~3 minutes
================================================================

STEP 1: Go to https://aistudio.google.com
STEP 2: Sign in with your Google account
STEP 3: Click "Get API key" in the top navigation bar
STEP 4: Click "Create API key"
STEP 5: Choose "Create API key in new project" (or select existing)
STEP 6: Copy the key — it starts with "AIza..."

Paste into .env as:
  GEMINI_API_KEY=AIzaxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Notes:
  - No credit card required
  - Free tier is more than enough for daily blogging
  - Model used: gemini-2.5-flash (auto-falls back to gemini-1.5-flash)
  - Key is tied to your Google account — keep it private


================================================================
  KEY 2: HF_TOKEN
  Service : Hugging Face (generates AI images via FLUX.1-schnell)
  Cost    : FREE — generous free tier for inference
  Time    : ~5 minutes
================================================================

STEP 1: Go to https://huggingface.co
STEP 2: Click "Sign Up" (top right) — use your email or GitHub
STEP 3: Verify your email address
STEP 4: Click your profile picture (top right) → "Settings"
STEP 5: In the left sidebar, click "Access Tokens"
STEP 6: Click "New token"
STEP 7: Fill in:
          Name: blog-automation
          Role: Read
STEP 8: Click "Generate a token"
STEP 9: Copy the token — it starts with "hf_..."

Paste into .env as:
  HF_TOKEN=hf_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Notes:
  - The token only needs "Read" access for inference API
  - Model used: black-forest-labs/FLUX.1-schnell
  - Free tier can be slow (5-15s per image) — that's expected
  - If model is loading, the script auto-waits and retries

  KEY 3-ALT: DEV.TO API KEY  [INSTANT — no waiting needed]
  Service : Dev.to (popular tech blog platform, growing fast)
  Cost    : FREE
  Time    : ~3 minutes
================================================================

If you can't get a Medium token, Dev.to is a great alternative.
It has a clean API and gives tokens instantly.

To use this with the project:
  - Add to .env as: DEVTO_API_KEY=xxxxxxxx
  - Tell the developer (or AI) to switch publisher.py to use Dev.to

Dev.to post API:
  POST https://dev.to/api/articles
  Header: api-key: YOUR_KEY
  Body: { "article": { "title": "...", "body_markdown": "...", "published": true } }

================================================================
  KEY 4: MEDIUM_AUTHOR_ID
  What    : Your unique Medium user ID (needed to post via API)
  Cost    : FREE
  Time    : 2 minutes (requires MEDIUM_TOKEN first)
================================================================

STEP 1: Make sure you have your MEDIUM_TOKEN from Key 3 above
STEP 2: Open your terminal
STEP 3: Run this command (replace YOUR_TOKEN with your actual token):

  curl -H "Authorization: Bearer YOUR_TOKEN" https://api.medium.com/v1/me

STEP 4: You'll get a JSON response like this:
  {
    "data": {
      "id": "1a2b3c4d5e6f...",   <-- THIS is your Author ID
      "username": "yourname",
      "name": "Your Name",
      ...
    }
  }

STEP 5: Copy the value of "id" (the long alphanumeric string)

Paste into .env as:
  MEDIUM_AUTHOR_ID=1a2b3c4d5e6f7g8h9i0j...

--- ALTERNATIVE: Use Hoppscotch (no terminal needed) ---

STEP 1: Go to https://hoppscotch.io (free, browser-based API tool)
STEP 2: Set method to GET
STEP 3: URL: https://api.medium.com/v1/me
STEP 4: Click "Headers" tab → Add header:
          Key: Authorization
          Value: Bearer YOUR_MEDIUM_TOKEN
STEP 5: Click "Send"
STEP 6: Copy the "id" value from the response

Notes:
  - This ID never changes — you only need to fetch it once
  - It's not the same as your username (it's a long internal ID)

  Cost    : FREE forever

Why this is needed:
  Medium's API only accepts public image URLs (https://...).
  It does NOT accept local file uploads. imgbb.com hosts your
  AI-generated images and gives back public URLs that Medium accepts.

STEP 1: Go to https://imgbb.com
STEP 2: Click "Sign up" (top right)
STEP 3: Register with your email (or Google/Facebook)
STEP 4: Verify your email
STEP 5: Once logged in, go to: https://api.imgbb.com
STEP 6: Your API key is shown on that page — copy it

Paste into .env as:
  IMGBB_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Notes:
  - Images are hosted permanently for free
  - No watermarks on images
  - API rate limit: plenty for our usage (a few images per blog)
  - If imgbb is down, the script continues without that image


================================================================
  KEY 6: TWITTER API KEYS  (4 keys total)
  Service : Twitter / X (share a catchy tweet after publishing)
  Cost    : FREE for posting to your own account
  Time    : ~15 minutes
================================================================

Why 4 keys?
  Twitter uses OAuth 1.0a for user-level actions (posting tweets).
  Think of it as two padlocks and two keys — the App lock (API Key +
  Secret) and your Account lock (Access Token + Secret). You need
  all four to post a tweet as yourself.

--- STEP A: Create a Developer Account ---

STEP 1: Go to https://developer.x.com  (formerly developer.twitter.com)
STEP 2: Click "Sign in" — use your existing Twitter/X account
STEP 3: Click "Sign up for Free Account"
STEP 4: Fill in your use-case description. Write something like:
          "Personal blog automation tool to auto-share new blog posts
          I publish. Will only post to my own account."
STEP 5: Agree to the Developer Agreement and submit
STEP 6: You should get immediate access (Free tier)

--- STEP B: Create a Project and App ---

STEP 7:  In the Developer Portal, click "+ Add App" or
         "Create Project" → give it a name like "blog-automation"
STEP 8:  Inside the project, click "+ Create App"
STEP 9:  Give the app a name: "blog-automation" (must be unique globally
         — add your name if taken, e.g. "blog-automation-karthik")
STEP 10: You will now see your API Key and API Key Secret — COPY BOTH
         immediately. They are shown only once.

Paste into .env as:
  TWITTER_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxx
  TWITTER_API_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

--- STEP C: Set App Permissions to Read + Write ---

STEP 11: In your app's dashboard, click "Settings" tab
STEP 12: Scroll to "User authentication settings" → click "Set up"
STEP 13: Fill in:
           App permissions    : Read and Write
           Type of App        : Automated App or Bot
           Callback URI       : http://localhost
           Website URL        : https://github.com  (or your own site)
STEP 14: Click "Save"

⚠️  IMPORTANT: After changing permissions, you MUST regenerate your
  Access Token and Secret (old ones carry old permissions).

--- STEP D: Generate Access Token and Secret ---

STEP 15: Go to your app's "Keys and Tokens" tab
STEP 16: Under "Authentication Tokens" → "Access Token and Secret"
         click "Generate"
STEP 17: Copy BOTH values — shown only once

Paste into .env as:
  TWITTER_ACCESS_TOKEN=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
  TWITTER_ACCESS_TOKEN_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Notes:
  - All 4 keys are tied to your Twitter account
  - Free tier: up to 1,500 tweets/month — more than enough
  - If you see a 403 error, your app permissions are wrong —
    repeat STEP C and regenerate tokens in STEP D
  - Never share these keys — anyone with them can tweet as you


================================================================
  KEY 7: LINKEDIN API KEYS  (2 keys total)
  Service : LinkedIn (share a professional post after publishing)
  Cost    : FREE
  Time    : ~25 minutes (OAuth flow is slightly involved)
================================================================

Why is LinkedIn more complex?
  LinkedIn uses OAuth 2.0, which means you go through a browser-based
  login flow once to get an access token. It's like "Sign in with
  LinkedIn" but the token is saved for your script to reuse.
  The token lasts 60 days, then you repeat the flow.

You will get 2 things:
  LINKEDIN_ACCESS_TOKEN  — your authentication token (expires in 60 days)
  LINKEDIN_PERSON_URN    — your unique LinkedIn user ID (never changes)

--- STEP A: Create a LinkedIn Developer App ---

STEP 1:  Go to https://developer.linkedin.com
STEP 2:  Click "Create App" (sign in with your LinkedIn account if needed)
STEP 3:  Fill in:
           App name       : blog-automation
           LinkedIn Page  : your LinkedIn profile URL
                            (e.g. https://www.linkedin.com/in/yourname/)
           App logo       : upload any small image (even a placeholder)
STEP 4:  Check the legal agreement box and click "Create app"
STEP 5:  You are now in the app dashboard. Go to the "Auth" tab.
STEP 6:  Note down your Client ID and Client Secret (you'll need them
         in the OAuth flow below)

--- STEP B: Request API Access (Products) ---

STEP 7:  Click the "Products" tab in your app dashboard
STEP 8:  Find "Share on LinkedIn" → click "Request access"
         (This gives you the w_member_social scope — needed to post)
STEP 9:  Find "Sign In with LinkedIn using OpenID Connect" → Request access
         (This gives you profile + openid scopes — needed to get your ID)
STEP 10: Both are usually approved instantly

--- STEP C: Set OAuth Redirect URL ---

STEP 11: Go back to the "Auth" tab
STEP 12: Under "OAuth 2.0 settings" → Authorized redirect URLs
         click "Add redirect URL"
STEP 13: Add exactly: http://localhost:8080/callback
STEP 14: Click "Update"

--- STEP D: Get Your Access Token (OAuth Flow) ---

STEP 15: Open your browser and paste this URL (replace YOUR_CLIENT_ID):

  https://www.linkedin.com/oauth/v2/authorization?response_type=code&client_id=YOUR_CLIENT_ID&redirect_uri=http://localhost:8080/callback&scope=openid%20profile%20w_member_social

STEP 16: Log in with your LinkedIn account and click "Allow"
STEP 17: Your browser will redirect to a URL like:
           http://localhost:8080/callback?code=XXXXXXXXXXXXXX
         The page will show an error (that's expected — localhost isn't
         running anything). Copy the CODE value from the URL bar.

STEP 18: Now exchange that code for an access token.
         Run this curl command in your terminal (fill in all YOUR_ values):

  curl -X POST https://www.linkedin.com/oauth/v2/accessToken \
    -H "Content-Type: application/x-www-form-urlencoded" \
    -d "grant_type=authorization_code" \
    -d "code=YOUR_CODE_FROM_STEP_17" \
    -d "redirect_uri=http://localhost:8080/callback" \
    -d "client_id=YOUR_CLIENT_ID" \
    -d "client_secret=YOUR_CLIENT_SECRET"

STEP 19: You will get a JSON response like:
  {
    "access_token": "AQX...",   <-- COPY THIS
    "expires_in": 5183944,
    "token_type": "Bearer"
  }

Paste into .env as:
  LINKEDIN_ACCESS_TOKEN=AQXxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

--- STEP E: Get Your LinkedIn Person URN ---

STEP 20: Run this command (replace YOUR_ACCESS_TOKEN):

  curl -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
       https://api.linkedin.com/v2/me

STEP 21: You will get a JSON response like:
  {
    "id": "abc123xyz",   <-- COPY JUST THIS VALUE (not the full URN)
    "localizedFirstName": "Your",
    "localizedLastName": "Name"
  }

Paste into .env as (just the ID, not the full URN):
  LINKEDIN_PERSON_URN=abc123xyz

Notes:
  - Your access token expires in ~60 days
  - To refresh: repeat STEP D only (app setup stays the same)
  - LINKEDIN_PERSON_URN never changes — save it permanently
  - If you get a 403 error, your "Share on LinkedIn" product
    access may still be pending — wait a few minutes and retry
